
# Day 11 - Processes and Process Management

## Objective

The goal for today was to understand what processes are in Linux, how to view them, and how to stop running programs.

---

## What I Learned

- I learned that a process is a running program in Linux
- I understood how to view processes using `ps` and `ps aux`
- I learned how to monitor system activity using `top`
- I learned how to stop processes using `kill` and `kill -9`

---

## What I Built / Practiced

- Viewed running processes using `ps` and `ps aux`
- Monitored system performance using `top`
- Created a process using `sleep`
- Identified and killed a process using its PID

---

## Challenges Faced

- Understanding process IDs (PID) initially
- Needed to be careful not to terminate important system processes

---

## Key Takeaways

- Every running program in Linux is a process
- Each process has a unique PID
- `top` provides real-time system monitoring
- `kill` is used to stop processes when needed
