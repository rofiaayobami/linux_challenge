# Day 16 - Passing Input to Shell Scripts

## Objective

The goal for today was to learn how to pass input into shell scripts and make them interactive.

---

## What I Learned

- Scripts can accept command-line arguments using `$1`, `$2`, etc.
- `$1` represents the first input, `$2` the second input
- The `read` command allows user input during script execution
- Scripts can be made dynamic and reusable using inputs

---

## What I Built / Practiced

- Created scripts that accept command-line arguments
- Used `$1` and `$2` to display input values
- Created interactive scripts using `read`
- Combined user input and script execution

---

## Challenges Faced

- Understanding how arguments are passed
- Remembering to include inputs when running scripts

---

## Key Takeaways

- Shell scripts can accept input from users
- `$1`, `$2` make scripts flexible
- `read` allows interactive input
- Input handling is essential for automation
