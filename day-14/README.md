
# Day 14 - Users, Groups and permission in Linux

## Objective

The goal for today was to understand how users and groups work in Linux and how permissions are managed.

---

## What I Learned

- Linux is a multi-user system
- Each user has a unique identity
- Users belong to groups
- Files have owners and permissions
- Permissions control access to files

---

## What I Built / Practiced

- Checked current user using `whoami`
- Viewed groups using `groups`
- Listed system users using `/etc/passwd`
- Checked file ownership using `ls -l`
- Learned how to change ownership using `chown`

---

## Challenges Faced

- Understanding the relationship between users and groups
- Interpreting permission formats
- I could not create or modify users because I do not have administrative (sudo) privileges on the server

---

## Key Takeaways

- Users and groups control access in Linux
- Every file has an owner and group
- Permissions define who can read, write, or execute
- Proper user management improves system security
