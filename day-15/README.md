# Day 15 - Introduction to Shell Scripting

## Objective

The goal for today was to understand the basics of shell scripting and how to automate tasks in Linux.

---

## What I Learned

- A shell script is a file containing Linux commands
- The `#!/bin/bash` (shebang) defines the script interpreter
- Scripts must be made executable using `chmod +x`
- Variables can store data for reuse
- Loops can automate repetitive tasks
- Conditional statements help make decisions in scripts

---

## What I Built / Practiced

- Created a script file using `touch`
- Edited scripts using `nano`
- Executed scripts using `./script.sh`
- Used variables to personalize output
- Used loops to process multiple files
- Used simple if statements to check file existence

---

## Challenges Faced

- Understanding script structure
- Remembering syntax for loops and conditions

---

## Key Takeaways

- Shell scripting helps automate repetitive tasks
- Permissions are required to execute scripts
- Variables and loops make scripts powerful
- Scripting is essential for data engineering workflows
