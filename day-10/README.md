
# Day 10 - File Permissions (Advanced)

## Objective

The goal for today was to gain a deeper understanding of file permissions in Linux and how to control access using numeric and symbolic methods.

---

## What I Learned

- I learned how to interpret file permissions using `ls -l`
- I understood the numeric representation of permissions (e.g., 7, 6, 5)
- I learned how to use `chmod` to modify permissions
- I practiced both numeric and symbolic permission settings

---

## What I Built / Practiced

- Created files and modified their permissions
- Used `chmod` with numeric values (e.g., 600, 777)
- Used symbolic mode (`u`, `g`, `o`) to adjust permissions
- Verified permissions using `ls -l`

---

## Challenges Faced

- Needed time to understand how numeric permissions are calculated
- Initially confusing to differentiate between user, group, and others

---

## Key Takeaways

- File permissions control access to files in Linux
- Numeric values (4, 2, 1) represent read, write, and execute
- `chmod` is essential for managing file security
- Avoid using `777` in real-world scenarios
